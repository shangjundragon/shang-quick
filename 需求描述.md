# RBAC 快速启动管理系统需求文档

## 1. 项目目标
基于 RBAC 模型的通用后台管理脚手架，提供用户、部门、菜单、角色及权限管理能力，实现前后端分离，开箱即用，只需少量修改即可扩展为具体业务系统。

## 2. 技术栈与约束
- **后端**：Go 语言，Gin 框架，自研 ORM 库 `github.com/shangjundragon/dbw`（AI 需通过阅读该库文档/代码了解其具体用法）。
- **前端**：Vue 3，Vue Router 4，Naive UI，Axios，推荐使用 TypeScript。
- **数据库**：MySQL 8.0。
- **请求方式**：**全部接口仅使用 `GET` 和 `POST` 两种 HTTP 方法**，不使用 PUT、PATCH、DELETE。
- **响应格式**：统一 JSON 结构，格式请参照项目中已有的 `backend/pkg/res_util/res_util.go` 的返回类型（例如包含 code、message、data 等字段，分页数据统一包装）。
- **认证方式**：JWT，登录成功返回 Token，后续请求在 Header 中携带。

## 3. 功能模块

### 3.1 用户认证
- **登录**：账号 + 密码登录，验证成功后返回 JWT Token 与用户基础信息（含角色编码、权限标识列表、可访问菜单树）。
- **密码加密**：使用 bcrypt。
- **鉴权中间件**：除登录接口外，所有 API 校验 Token 有效性，提取用户 ID、角色、权限标识并注入上下文。

### 3.2 用户管理
- 分页查询用户列表，支持按用户名、手机号、状态、部门筛选。
- 新增用户（可分配角色、部门，设置初始密码）。
- 编辑用户基本信息（不包含密码）。
- 启用/禁用用户状态。
- 重置用户密码（管理员操作）。
- 逻辑删除用户。
- 一个用户属于一个部门，可拥有多个角色。

### 3.3 部门管理
- 查询所有部门，以树形结构返回。
- 新增部门（指定上级部门、名称、排序）。
- 编辑部门信息。
- 删除部门前校验：若存在子部门或被用户引用，禁止删除并给出提示。

### 3.4 菜单管理
- 菜单类型分为三类：**目录**（一级分组，无实际路由）、**菜单**（对应前端路由和页面）、**按钮**（权限标识，控制页面内操作）。
- 以树形结构维护所有菜单项，支持增删改查。
- 字段包括：父级菜单、名称、类型、路由路径（目录/菜单）、前端组件路径、图标、排序、是否外链、是否隐藏、权限标识（按钮必填）。
- 删除菜单时级联删除所有子菜单。
- 前端根据后端返回的菜单树动态生成路由，按钮通过权限指令控制显隐。

### 3.5 角色管理
- 分页查询角色列表。
- 新增/编辑角色（名称、标识编码、描述、状态）。
- 为角色分配菜单权限：传入菜单 ID 列表（包含目录、菜单、按钮），后端全量替换该角色拥有的菜单关联。
- 查看某角色已分配的菜单 ID 列表（用于分配时回显）。
- 删除角色前校验：若存在用户引用，禁止删除。
- 预留超级管理员角色：当角色标识为 `admin` 时，拥有全部权限，无需显式分配菜单。

### 3.6 权限控制
- **前端**：登录后获取当前用户的权限标识集合及菜单树，动态注册路由，菜单和按钮根据标识显隐。
- **后端**：每个接口可配置所需权限标识（如 `user:add`），权限中间件检查用户是否拥有该标识，`admin` 角色直接放行，否则返回 403。
- 权限判断可基于用户所有角色的权限标识并集。

### 3.7 个人中心
- 查看并修改当前登录用户的基本资料（如昵称、手机号、邮箱等，不含密码）。
- 修改自身密码（需验证旧密码）。

### 3.8 操作日志（可选，作为默认小模块）
- 记录用户的关键操作（登录、用户增删改、角色权限变更等），包含操作人、IP、时间、操作内容。
- 提供分页查询日志的接口，用于审计。

## 4. 交互与接口约定
- 接口路径统一使用 `/api/v1/` 前缀。
- **所有接口均采用 GET 或 POST 方法**：
    - 获取类（查询、详情）使用 GET，参数通过 query string 传递。
    - 新增、修改、删除、状态切换等操作统一使用 POST，参数通过 JSON Body 传递。
- 返回格式参照 `backend/pkg/res_util/res_util.go` 定义


## 5. 前端路由与页面结构
- **固定路由**：登录页 `/login`。
- **主布局**：左侧动态菜单，右侧内容区域。
- **内置页面**：
    - 仪表盘 `/dashboard`（欢迎页）。
    - 系统管理分组 `/system` 下：用户管理、部门管理、菜单管理、角色管理、操作日志。
    - 个人中心 `/profile`。
- 动态路由实现：请求 `POST /api/v1/user/menus`（示例）获取用户菜单树，前端转换为 Vue Router 配置，通过 `addRoute` 动态添加。

## 6. 数据库相关说明
- 默认使用 sqlite 存储，全部表由 `dbw` ORM 管理，定义模型结构体。
- 用户、部门、菜单、角色之间存在关联关系，核心表应包括：用户表、部门表、菜单表、角色表、用户-角色关联表、角色-菜单关联表。
- 支持软删除。

## 7. 开发与交付要求
- AI 需阅读 `github.com/shangjundragon/dbw` 的使用方式，用其完成数据初始化、查询、关联等操作。
- 初始化 SQL 或种子数据需包含超级管理员账号 `admin / admin123`，以及一组基础菜单和角色。
- 提供完整的后端代码（入口、路由、中间件、控制器、模型、权限校验）和前端代码（登录页、布局、各管理页面、Axios 封装、权限指令），确保前后端联调可直接运行。
- 代码风格简洁清晰，关键逻辑添加注释。
- 暂不实现数据字典、系统配置等扩展功能，仅完成上述核心闭环。
